搭建 Hexo 轻博客

Posted on 2018-03-02 | In Web , hexo

搭建博客的第一步，就是要找到符合自己需求的博客框架。之前用 LAMP 搭过 Wordpress，但始终觉得就写博客这个需求来说太重了，环境尽量越简单越好。与同事们交流得知 Hexo，自己便开始 DIY。

环境准备

  • Node.js
  • 安装 hexo npm 包
  • git bash 客户端

初始化

初始化 hexo 目录有两种方式：

  • 首先创建目录，在目录中运行 cmd.exe 并执行 hexo init，该命令会将当前文件夹作为目录
  • 执行打开 cmd 或 terminal，使用 hexo init <folder> 初始化目录，并使用 cd <folder> 导航到该目录

    hexo init <folder> 命令中，如果 folder 所包含的目录层级中有任何不存在的目录，该命令会自动创建该层级下的目录

新建完成后，得到以下的目录结构：

_config.yml

_config.yml 包含了所有对 hexo 博客站点的配置信息。

网站(Site)

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 作者名字
language 网站使用的语言
timezone 时区

其中，description 主要用于 SEO，告诉搜索引擎一个关于站点的简单描述，通常建议在其中包含网站的关键词。author 参数用于主题显示文章的作者。

网址(url)

参数 描述 默认值(示例值)
url 根地址 http://www.your-site.conm/blog/
root 站点根目录 /blog/
permalink 文章的永久链接格式 :year/:month/:day/:title/
permalink_defaults 永久链接中各部分的默认值

url 和 root 总是成对修改，例如，想要把站点的根目录放在子目录 /blog 下，那么 root 的值应该修改为 /blog/，url 的值对应修改为 http://www.your-site.com/blog/。
官方文档中的网站存放于子目录的意思是，hexo 项目包含的是生成站点的源代码文件，包括 source 目录下的 _posts，均用于生成网站，而“网站”是指生成好之后的静态资源文件集。

更多配置详情参考 https://hexo.io/docs/configuration.html

部署到 Github Pages

当站点准备就绪后，下一步就是要给它找个 host，github 为每个账号提供了 host 站点的服务——Github Pages，在 github 中创建一个 username.github.io 的仓库，然后将资源文件 push 到该仓库，再访问唯一的 https://username.github.io/ url 即可。

hexo 提供了自动部署机制，在 _config.yml 中找到 deployment 这一项：

部署的具体文档参考 https://hexo.io/docs/deployment.html

Github Pages 同时支持自定域名解析个人站点，假设已经有了一个域名 example.com，那么在 dns 提供商后台将喜欢的二级域名或 www 域名指向 username.github.io 即可，具体参考 https://help.github.com/articles/using-a-custom-domain-with-github-pages/

使用自定域名唯一的缺点是无法使用 https。
