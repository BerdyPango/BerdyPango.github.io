利用 DDNS 实现内网穿透

发表于 2018-05-23 | 分类于 Home Server
| 字数统计: 602
本文介绍了 DDNS 的背景及原理，以及如何使用开源手段实现 DDNS

背景

众所周知，由于 IPv4 资源的逐渐枯竭，国内的家庭网络 IP 地址是由 ISP(Internet Server Provider) 动态分配的，这使得家庭服务器无法使用固定 IP 地址向外提供服务，于是 DDNS(Dynamic Domain Name System) 便派上了用场。

原理

利用家庭主机不断检测本地网络的公共 IP 地址变化，一旦变化便向指定的域名更新其记录值，这样当从外网通过域名解析 IP 地址时，会始终指向同一个主机。

前提条件

要实现 DDNS，至少应该准备:

  • 可远程管理的已注册域名
  • 可配置端口转发的路由器
  • 确保本地网络的公网 IP 可由外网访问

上述第三条并不是硬性的技术指标，而是近年来有些 ISP 为了腾出公网 IPv4 资源，在没有告知用户的情况下将家庭网络置于某个区域的局域网之下。如果用户没有从外网访问家庭网络的需求，这一做法也无伤大雅，否则，一定要确认这一点，并向 ISP 申请开放公网 IP 地址。

实现

国内市场上有诸如花生壳，国外市场上有诸如DuckDns 的 DDNS 服务商，他们为用户提供二级域名使用权和相应的客户端软件用以更新 DNS 记录。为了争取最大的自由度，本文将绕过第三方通过开源手段来实现。

假设我在阿里云(也可能是腾讯云，重要的是其提供相应的域名管理 API)注册了 example.com，阿里云负责确保全网都可访问该域名。然后在家庭网络中的某台主机安装 Git，从 ddns-client-aliyun 拉取用于更新阿里云域名记录的 ddns 客户端，配置 ha.example.com 并运行一次。

配置路由器端口转发及防火墙设置

由于众所周知的原因，国内 ISP 屏蔽了家庭网络的 80 及 443 端口，我们需要选择其他端口作为访问 Http(s) 服务的端口并且在入站防火墙中放通这些端口的 TCP 协议。最后，提供 Http(s) 服务的家庭网络的主机通常位于路由器之后，所以需要在路由器配置端口转发。
